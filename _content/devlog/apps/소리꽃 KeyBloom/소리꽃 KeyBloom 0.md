## 요약

MIDI 파일을 넣으면 하단 88건반이 타이밍에 맞춰 눌리고, 그 건반 위치에서 파티클 이펙트가 피어오르는 연주 이펙트 비주얼라이저. 피어오름·불꽃·토네이도·분수 같은 파티클 움직임과 색·모양·광택을 곡 전체 전역 파라미터로 조절하고, 결과를 16:9 영상으로 저장한다. 서버 없이 브라우저 안에서 완결되는 정적 사이트.

---

## 제작 동기

피아노 연주 영상을 만들 때 건반 위로 이펙트가 피어오르는 비주얼을 직접 만들고 싶었다. 실질적으로 원하는 건 실시간 연주 — 건반을 치는 그 순간 해당 건반 위로 이펙트가 피어오르는 것이다. 다만 실시간은 상태가 복잡해지니, 먼저 미리 만들어 둔 MIDI를 타임라인으로 재생하며 이펙트를 입히는 버전부터 만들었다. 비주얼 엔진(건반·파티클·이펙트)을 파일 재생으로 먼저 다져 두고, 나중에 노트 입력만 실시간으로 바꿔 얹는 순서가 안전하다고 봤다. 기존에 쓰던 TypeScript + Vite + Canvas 조합으로 새 인프라 학습 없이, 서버 없는 정적 사이트로 완결하는 걸 목표로 했다.

---

## 목표 설정

- 작업 기간: 2026.07.03. ~ 진행 중
- 최종 목표
    - 실시간 연주 모드 — Web MIDI로 라이브 건반 입력을 받아, 치는 순간 그 건반 위로 이펙트가 피어오르게
- 1단계(MVP, 완료): 비주얼 엔진을 파일 재생으로 먼저 완성
    - MIDI 파일 재생 + 88건반 눌림 동기화 + 건반 위 파티클 이펙트
    - 이펙트를 곡 전체 전역 규칙(파라미터)으로 조절, 값 바꾸면 미리보기 즉시 갱신
    - 파티클 움직임·색·모양·광택 조절 + 16:9 영상 저장

---

## 주요 작업

1. 환경 셋업 (Vite + TypeScript + Canvas 2D) ✅
2. MIDI 파싱(@tonejs/midi) + 여러 트랙 병합/선택 ✅
3. 88건반 렌더 + 재생 타이밍에 눌림 동기화 ✅
4. 파티클 시스템 — 데이터 주도 모션(피어오름·방사형 개화·상승 후 흐트러짐·나선형 토네이도·불꽃·분수) ✅
5. 색(velocity 편향 랜덤 lerp)·모양(다중 선택 랜덤)·광택(스페큘러 + 가산 글로우) ✅
6. 오디오 — 사용자 오디오 파일 싱크 재생, 없으면 합성음 ✅
7. 스프라이트 아틀라스로 대량 파티클 렌더 최적화 ✅
8. 영상 저장 — canvas.captureStream + MediaRecorder로 WebM 추출(해상도·배경·오디오 옵션) ✅
9. 성능 최적화 — 밀집 구간 버벅임 개선(두 모드 공통 기반) ⏳
10. 큐(Cue) 기능 — 구간별 색·모양 오버라이드(프리셋/트리거 분리 → 실시간과 공통) ⏳
11. 실시간 연주 모드 — Web MIDI로 라이브 건반 입력, 실질적 최종 목표 ⏳
12. 건반 눌림 색 지정 — 지금은 하드코딩(하늘색), 사용자가 고르게 ⏳
13. velocity 연동 대상 선택 — 개수·크기·속도·높이·선명도·광택에 반영 여부 ⏳
14. Cloudflare Pages 배포 ⏳

- MVP 코어 완성. 성능 최적화(버벅임)와 큐(구간별 색·모양 오버라이드)는 다음 단계로 남김
- 실질적 목표인 실시간 연주 모드는, 지금 만든 비주얼 엔진(건반·파티클·이펙트)을 그대로 두고 노트 입력 소스만 MIDI 파일 → Web MIDI로 바꿔 얹는 방향으로 확장 예정. 완전히 다른 앱이 아니라 입력층 교체다
