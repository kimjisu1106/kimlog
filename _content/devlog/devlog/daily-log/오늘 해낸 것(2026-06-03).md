## 배운 점

- 지난번엔 포켓몬 도감완성 게임을 React로, 이번엔 런 게임을 JS로 진행했다. React로, Vanilla JS로 게임을 만들어보니 충돌 감지·씬 전환·물리 같은 것을 직접 구현해야 하는 구조였다. 게임 전용 프레임워크인 Phaser를 쓰면 이런 것들이 내장되어 있어서 게임 로직에만 집중할 수 있다는 걸 알게 됐다.
	- React, Vanilla JS로 할 때의 단점
		- Claude Code한테 "충돌 감지 만들어줘", "씬 전환 만들어줘" 매번 요청해야 하며 일일히 컴포넌트화 해달라고 해야됨 → 크레딧 소모됨
		- 게임이 복잡해질수록 기존 코드와 충돌 가능성 높아짐 → 디버그 요청 늘어남
		- 게임 로직 구현에 집중하기 어렵고 인프라 코드가 많아짐
	- Phaser로 할 때
		- "캐릭터 점프 추가해줘" 하면 Claude Code가 Phaser API 써서 짧고 안정적으로 구현
		- 씬 관리, 물리, 충돌이 검증된 코드라 버그 가능성 낮음
		- 결과적으로 대화가 짧아지고 크레딧 절약
