## 오늘 한 일

- 소리꽃 KeyBloom — 실질적 최종 목표였던 실시간 연주까지
	- 실시간 연주 모드 — Web MIDI 건반이나 컴퓨터 키보드로 라이브 연주. 치는 순간 그 건반 위로 이펙트. 비주얼 엔진은 그대로 두고 노트 입력 소스만 교체(`noteOn`/`noteOff` 공통 핸들러). 숫자키 1~6로 큐 전환, 라이브 수동 녹화
	- 오디오 지연 개선·진단 — 최소 버퍼·콜드스타트 방지·빠른 어택. 실측 지연을 화면에 표시
	- 프로젝트 저장/불러오기 — 큐·큐포인트·트랙·원본 MIDI(base64)를 JSON 한 파일로. 새로고침해도 안 날아감
	- 라이브 저지연 연주는 웹 한계(브라우저가 ASIO를 못 씀)라, 추후 네이티브(하이브리드)로 가기로 방향 결정
- 메트로놈(Simple Metronome) — 예전에 ChatGPT 복붙으로 만든 앱을 Claude Code로 관리 시작
	- git이 없던 프로젝트라 저장소를 새로 만들고, 서명 키스토어 비밀번호가 평문으로 노출돼 있던 걸 분리해서 보호
	- 프로젝트 지침(CLAUDE.md) 작성 — 다른 프로젝트들의 작업 원칙을 이 앱에 맞게 정리
	- 강아지 테마 1차 — 밋밋하던 비트 인디케이터를 발자국과 강아지 얼굴로 바꾸고, 기본(다크) ↔ 강아지(크림·밤색) 테마 전환 토글을 붙임

## 어려웠던 점

- 실시간 연주에서 소리가 늦게 나서 지연을 줄이려 했는데, 실측해보니 장치 지연 48ms가 브라우저 바닥이었다. 브라우저는 오디오 인터페이스의 저지연 드라이버(ASIO)를 못 쓰고 OS 공유모드로만 소리를 내서, 좋은 장비를 꽂아도 웹에선 저지연이 안 나온다는 걸 숫자로 확인했다. 앱에서 더 손댈 수 없는 한계라 네이티브 얘기로 넘어감
- 메트로놈 — 밋밋한 UI를 귀엽게 바꾸려니 방향 잡는 게 제일 고민이었다. 강아지 테마에 맞게 메트로놈의 똑-딱 왕복을 강아지가 발자국 컨셉으로 정했다.

- (소리꽃 KeyBloom TIL 9로 정리) 파일 재생과 실시간 연주를 `noteOn`/`noteOff` 공통 핸들러로 합쳐 비주얼 엔진 재사용 / Web MIDI로 건반 입력(`requestMIDIAccess`, 메시지 파싱) / 컴퓨터 키보드를 건반으로(Musical Typing, 오토리핏·옥타브 처리) / 웹 오디오 지연의 정체(WASAPI 공유모드 vs ASIO) / 전체 상태를 파일 하나로 저장(직렬화 + MIDI base64 + 후방호환 3원칙)
- (안드로이드 TIL로 정리) 안드로이드 앱 서명 키스토어·비밀번호를 git에서 빼는 법(keystore.properties + gitignore) / 앱 안에서 테마를 통째로 바꾸는 법(setTheme는 setContentView 전에, recreate로 재적용) / 색 하드코딩 대신 커스텀 테마 속성(?attr)으로 참조 / 벡터 드로어블 + backgroundTint 리틴트로 인디케이터 점등
