	- HUD에 완성 목표 썸네일과 지금 부을 수 있는 색 힌트 추가

	- 앞으로 할 레벨은 도안·난이도를 숨기고 ??? 로. 난이도 배지는 어려움·매우 어려움에만
	- 들어가기 전 완성할 그림을 크게 보여주는 프리뷰(탭하면 스킵)

	- 시작 코인 0 + 일일 접속 랜덤 보상 기획. 상점(번들·NO ADS·코인팩·아이템) 화면 추가, 결제는 나중
- 모래게임 Sandrop — 팝업·목록 손보기
	- 승리 팝업: 폭 1.5배, 광고 보고 코인 2배가 먼저 뜨고 다음 레벨은 1초 뒤에, 우상단 X
	- 레벨 선택: 한 번에 너무 많이 보여서 아이콘·글씨·버튼을 키워 위아래 2개씩만 보이게
- 소리꽃 KeyBloom — 파티클 모션 2종 추가
	- 나선 기둥 — 토네이도보다 좁은 반경이 퍼지지 않고 원통형으로 회전하며 더 높이 상승
	- 물고기 유영 — 좁은 S자 꼬리짓 + 꼬리짓에 박자 맞춘 추진(burst-glide). 물고기마다 좌/우 방향을 정해 그쪽으로만 헤엄침
	- 튜닝 — 회전 속도 2배, 좌우 퍼짐 확대, 방향 확정형(가운데 직진 없음)

## 어려웠던 점

- 디자인 제안을 어디까지 받을지 자르는 판단 — 예뻐 보이는 팔레트를 도안에까지 밀면 게임의 콘텐츠가 뭉개진다. 샘플을 직접 눈으로 보고 나서야 선을 그을 수 있었다
- 하트와 코인의 역할이 겹쳐 있어서 게임오버 화면이 헷갈렸다. 원하는 흐름을 말로 설명하기 어려워 화면을 직접 그려서 전달했다

## 배운 점

- (모래게임 Sandrop TIL 6으로 정리) 스킨과 콘텐츠는 팔레트를 나눠 써야 한다 / 오프라인 전제면 폰트는 번들 + 서브셋 / 긴 목록은 보이는 것만 지연 로드 / 테스트에서 에셋 I/O는 가짜 저장소로 걷어낸다
- (소리꽃 KeyBloom TIL 11로 정리) 데이터 주도 모션에선 새 모션 = 물리값 질문(토네이도와 원통 기둥의 차이는 진폭 성장 파라미터 하나) / 물고기 burst-glide는 수직 추진 맥동을 꼬리짓과 위상 동기로 / 군집의 형태는 위상 분산이 만든다 / 방향 있는 랜덤은 부호·크기 분리 + 최소값 보장
