## 배운 점

1. Gemfile: Ruby 프로젝트의 의존성 목록. Jekyll이 Ruby로 만들어져 있어서, 사이트를 빌드하려면 어떤 버전의 Jekyll과 플러그인을 쓸지 명시해줘야 한다. GitHub Pages는 Gemfile 없이도 자체적으로 Jekyll 버전을 알아서 처리했는데, Netlify는 직접 빌드하니까 이 파일이 필요하다.
2. `kramdown-parser-gfm`: GitHub Flavored Markdown(GFM) 파싱 라이브러리. 체크박스(`- [ ]`), 표(`|---|`), 코드 펜스(` ``` `) 같은 마크다운을 Jekyll이 처리하려면 이 gem이 필요. GitHub Pages는 기본으로 이걸 내장하고 있어서 따로 선언 하지 않아도 되었으나 Netlify는 직접 빌드하니까 Gemfile에 명시해줘야 한다.
