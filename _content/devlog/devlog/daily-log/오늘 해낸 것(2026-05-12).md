## 오늘 한 일

- 회사 툴
	- `serve` 함수 수동 적용
		- 이유: `DEBUG=False`를 했더니 static file 서빙이 중단되어 JS가 작동하지 않음.
		- 원인: 프레임워크의 의도적인 설계(`DEBUG=False`는 프로덕션 상태니까 Nginx 등 웹서버에 위임) 때문
		- 해결: Nginx의 도입이 속도(C로 만들어진 웹서버)나 구조(static/ 폴더를 Nginx가 직접 읽으므로 `serve()`등록 안해도 됨) 면에서 이점이 있으나 현재의 환경(적은 트래픽, 유지보수 최소화)에서는 유의미하지 않다고 판단함.
	- `FK` 오류 수정
		- 이유: 전월, 당월, 익월 데이터가 유기적으로 맞물리는 model에 FK 시점이 안맞는 것을 뒤늦게 발견. 전월 자료를 불러와야 하는 곳에 전전월이 불러와짐.
		- 원인: 기존 시스템에서 가져온 DB는 시점이 맞춰져 있었기 때문에 에러가 숨겨져있었음. 이번달에 처음으로 새 시스템에서 DB를 생성하며 드러남.
		- 해결: FK 시점 맞추기. 다행히 DB는 이번달거만 수정하면 된다.
	- HiddenInput 파싱 에러 수정
		- 이유: 특정 상황에서는 `<form>`안의 `<input>`에 자동 계산된 숫자를 넣어야하는 상황이 생김. 사람이 볼 때는 comma가 필요하고, 서버에는 숫자로 전송해야하는 상황. 
		- 원인: comma가 포함된 숫자가 문자열로 서버에 전달되어 cannot be parsed 에러 발생.
		- 해결: HiddenInput에 정수, 화면 표시는 별도 `<span id="w_total_display">`로 분리.

## 어려웠던 점

- 문제점은 발견되었는데 그 원인이 `FK` 시점이 안맞는 것 및 오기입된 DB로 인한 문제점이라는 걸 발견하기까지 오래걸렸다. 왜냐면 기존 것은 기존 DB에서 sync했으니 전부 멀쩡한데 이번에 생성한 것만 이상한 상황이었기 때문에 무슨 숫자를 어디서 갖고온건지 역추적해야했다. 계산식을 하나하나 살펴보며 디버깅.
