## 어려웠던 점

- "보호 기간 = 안 아픔"을 그냥 레벨 0으로 만들면 보호가 끝나는 순간 갑자기 아픔으로 점프한다. 배고픔 단계(1~2)는 그대로 보여주고 아픔(3)만 막는 레벨 캡 방식으로 풀어서, 상태 변화가 자연스럽게 이어지게 함
- 광고 ID가 앱 ID(~)와 단위 ID(/) 두 종류라는 걸 구분해야 했다 — 매니페스트에는 앱 ID, 코드에는 단위 ID. 단위 ID만 실 ID로 바꾸면 매니페스트가 테스트 앱 ID라 실광고가 안 나옴
- Crescendo 함선 — 도형 몇 개를 겹쳐놓는 것만으로는 "비행선"으로 안 읽혔다. 실루엣의 외곽 형태 자체가 정보를 전달해야 해서, 뾰족한 기수·함교·행어 노치 같은 특징 요소를 폴리곤으로 직접 그리니 그제서야 함선으로 보임
- 메트로놈 UI 수치는 한 번에 안 맞는다 — 박자표 크기·발자국 크기·버튼 줄바꿈을 Android Studio에서 눈으로 확인하며 여러 번 조정했다. BPM 숫자를 키웠더니 옆 ± 버튼이 좁아져 글자가 줄바꿈되는 식으로, 한 요소를 키우면 이웃이 밀리는 상호작용이 계속 나왔다
- 발자국 줄바꿈이 세트를 찢는 문제 — 점 개수로 반씩 나누니 9/8에서 큰 발자국과 그 작은 발자국들이 다른 줄로 갈라졌다. "한 박 = 한 세트"라는 단위를 레이아웃에도 그대로 반영해야 했다
- 블로그 분석 도구가 GTM·GA·Cloudflare Insights 3중으로 겹쳐 보였는데, 뜯어보니 GA 스니펫은 설정값이 없어 아예 출력되지 않는 죽은 코드였다. 실제 활성은 GTM + Cloudflare Insights 2개 — GTM 컨테이너 안에 GA4 태그가 있는지는 대시보드에서 확인해야 함
- 제목에 날짜를 붙이니 목록에서 날짜가 이중 표시되는 문제 — 목록 표시용 short_title 필드를 따로 두고 default 필터로 fallback시켜 해결
- Lumior — 무엇을 타임라인에 넣을지가 제일 고민이었다. 연호·Year가 빈 템플릿을 넣으면 설정 문서까지 섞이고, 빼면 정작 "연호 없음"을 짚어야 할 점검이 무력해진다. 점검 취지를 살리려 "값이 비어도 필드가 있으면 포함"으로 정했다. 해설집은 억지 유사 매칭 대신 원본과 완전히 일치하는 것만 날짜를 채우기로 선을 그었고, 인물축은 이리스 0년에 스토리가 몰려 년 단위론 인물 교차가 안 보여 년/월/일 구분 단위를 고르게 했다
