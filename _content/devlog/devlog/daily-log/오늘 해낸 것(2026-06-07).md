## 어려웠던 점

- 캐릭터 스프라이트를 Gemini로 생성했는데 프레임 간 일관성 유지, 팔다리 겹침 등 수정 불가한 이슈가 생겼다.
	- 해결방법: 3D 렌더 → 낱장 프레임 애니메이션 파이프라인 구축
		- VRoid에서 캐릭터 생성 → Blender import(`*.vrm`) & export(`*.fbx`→ Mixamo로 재리깅 + 모션(With Skin, In Place) → Blender 재 import → 측면 직교 카메라 + 라이팅 + Rendered 미리보기 → 해상도/Sensor Fit 세팅 → 프레임 단위로 PNG Export
		- 스프라이트 시트로 추가 편집하지 않고 Phaser에서 낱장 프레임으로 애니메이션 구축
	- 이유: 일일이 도트를 찍는 비용 절감. 
	- 한계: VRoid에서 제공되는 디자인 내에서 고르거나 VRoid와 호환 가능한 Asset을 별도로 제작해서 사용해야 함. 사람 형태가 아닌 캐릭터는 구현이 불가함.

## 배운 점

- 캐릭터 스프라이트 생성하는 것 때문에 별도로 도구를 제작해야 하나 고민했는데 그렇게 안하고 해결 가능한 방법을 찾아냄.
	- 도구는 게임에서 반복이 실제로 쌓인 뒤에 추출하는 것 — 미리 범용 도구부터 지으면 오버엔지니어링
	- 8등신 + 변주 많은 캐릭터는 2D 레이어로 만들면 (액션 × 프레임 × 부위) 곱셈 노가다이므로 3D 본 애니메이션 한 세트 재사용으로 커버 가능하다
	- Rokoko 리타겟 애드온은 Blender 4.4+의 새 애니메이션 시스템과 비호환이라 크래시 발생되어 사용할 수 없다(구버전이나 우회 필요).
	- VRoid는 MToon(toon) 셰이더라 FBX/Mixamo를 거치면 텍스처가 날아감 → 색은 원본 모델에서 입히고, Mixamo한텐 모션만 맡긴다
	- Mixamo 다운로드: Without Skin = 뼈대만, With Skin = 메시 포함
	- 이미 Mixamo 본으로 재리깅한 모델이면 Mixamo에서 모션을 직접 받으면 됨 (리타겟 불필요)
- 발차기의 경우 가로가 길기 때문에 Orthographic Scale을 Vertical로 변경해야 한다. Sensor Fit은 긴 축 기준.
- Rendered 뷰포트 셰이딩 = 실시간 미리보기 / 출력 해상도가 작으면 실제 렌더가 픽셀처럼 나옴
