## 설계 원칙

### 규칙과 겉모습을 분리한다 — 엔진 순서는 그대로, 화면만 흩뜨린다

캔버스는 매 틱 몇 픽셀씩 채워진다(메터드 붓기). 그런데 엔진이 "최하단 행부터, 같은 행은 왼쪽부터" 순서로 채우니까, 화면에서는 물이 차오르듯 가로줄이 한 줄씩 쓸려 올라갔다. 모래가 알알이 떨어지는 느낌이 아니었다.

가장 쉬운 해법은 엔진의 채움 순서를 랜덤으로 바꾸는 것이다. 하지만 그 순서는 시뮬레이터·봇이 "이 레벨이 클리어 가능한가"를 판정하는 규칙과 얽혀 있어서(Dart·Python 공통 fixtures로 검증), 건드리면 파리티가 깨진다.

그래서 순서는 진실로 두고, 화면에만 흩뜨리는 층을 얹었다. 붓힌 픽셀마다 "탄생 시각"을 랜덤 지연으로 기록해두고, 아직 그 시각이 안 됐으면 잠깐 미채움 상태로 그린다.

```dart
// pourTick으로 이번 틱에 채워진 픽셀 → 0~0.15초 뒤 등장하도록 예약
for (final i in poured) {
  _fillBorn[i] = _animT + _rng.nextDouble() * 0.15;
}
```

```dart
// 페인터: 아직 탄생 시각 전이면 미채움으로 둬서 스캔라인 순서를 흩뜨린다
if (now < born) {
  c = Color.lerp(full, AppColors.guideBlend, 0.80)!;
}
```

"쌓이는 순서는 규칙대로, 눈에 보이는 타이밍만 흩어지게" — 로직과 연출을 분리하니 규칙을 안 건드리고도 질감이 완전히 달라졌다.

## 모래 질감 (CustomPaint 시각 레이어)

### 등장 플래시 — 픽셀이 처음 뜰 때 흰색에서 본색으로

탄생 시각이 지나면 그 픽셀을 잠깐 흰색에서 본색으로 lerp해 "톡 튀며 안착"하는 느낌을 준다. lerp(선형 보간)는 두 색 사이를 정해진 비율만큼 섞어 중간색을 얻는 것이다. 0.18초짜리 짧은 플래시다.

```dart
final t = ((now - born) / 0.18).clamp(0.0, 1.0);
if (t < 1) c = Color.lerp(Colors.white, c, 0.35 + 0.65 * t)!; // 흰색 → 본색
```

### 픽셀마다 밝기를 살짝 다르게 — 매 프레임 변하면 안 되니 고정된 랜덤으로

같은 색이라도 픽셀마다 밝기가 미세하게 다르면 알갱이처럼 보인다. 여기서 `Random`을 쓰면 매 프레임 값이 바뀌어 화면이 지글지글 떨린다. 그래서 셀 인덱스를 해시해(픽셀 번호를 뒤섞어 그 픽셀만의 고정된 숫자를 뽑아) 결정적으로 밝기를 정했다 — 같은 픽셀은 언제 그려도 같은 밝기라 안 흔들린다.

```dart
final h = (i * 48271) & 0x7; // 셀 인덱스 해시 (0~7, 결정적)
final c = h < 4
    ? Color.lerp(full, Colors.black, h * 0.015)!   // 살짝 어둡게
    : Color.lerp(full, Colors.white, (h - 4) * 0.02)!; // 살짝 밝게
```

연출은 프레임마다 다시 그려지므로, "매번 달라 보이면 안 되는" 값은 랜덤이 아니라 인덱스 기반 결정적 함수로 만들어야 한다.

## 파티클

### 모래 줄기 — 좌우 sway + 착지 스플래시

붓는 중인 바스켓에서 캔버스 목표 픽셀로 떨어지는 모래 줄기를, 베지어 경로 위에 알갱이를 흩뿌리고 각 알갱이를 sin으로 좌우로 흔들어 표현했다. 착지 지점에는 잔입자가 위로 튀어 오르는 스플래시를 더했다.

```dart
const grains = 12;
for (var k = 0; k < grains; k++) {
  final u = ((k / grains) + phase) % 1.0;
  final p = _bezier(s.from, ctrl, s.to, u);
  final sway = math.sin(u * 34 + k * 2.1) * 1.4; // 알갱이마다 좌우 흔들림
  canvas.drawCircle(Offset(p.dx + sway, p.dy), 1.4 + 1.2 * u + (k & 1) * 0.5, dot);
}
// 착지 스플래시 — 목표점에서 위로 튀어오르는 잔입자
for (var k = 0; k < 4; k++) {
  final a = (phase * 2 + k / 4) % 1.0;
  final dy = -2.0 - 4 * a; // 위로
  canvas.drawCircle(s.to + Offset(/*좌우*/, dy), 1.3 * (1 - a) + 0.4, splash);
}
```

## 레이아웃

### 캔버스는 "화면 정중앙"이지 "남는 영역 정중앙"이 아니다

캔버스를 엘리베이터를 제외한 영역 기준으로 가운데 놓았더니 오른쪽으로 ~18px 치우쳤다. 왼쪽에 엘리베이터가 폭을 먹고 있어서, "남은 폭의 중앙"은 화면 중앙이 아니었던 것.

좌우 여백을 엘리베이터 폭만큼 대칭으로 잡아 화면 전체 기준 정중앙에 놓았다. 폭은 좌우 여백(엘리베이터 폭)을 뺀 값으로 제한해 좌측 라이저 타일과 닿지 않게 했다.

```dart
// ❌ 엘리베이터 뺀 영역 기준 → 오른쪽으로 치우침
// ✅ 화면 전체 기준, 좌우를 엘리베이터 폭으로 대칭
final canvasSide = math.min(w - 2 * (elevW + 4), bandH);
final canvasRect = Rect.fromLTWH((w - canvasSide) / 2, /* y */, canvasSide, canvasSide);
```

한쪽에만 고정 요소(사이드바·엘리베이터)가 있으면 "정중앙"의 기준을 화면 전체로 잡되, 반대쪽 여백을 그 폭만큼 맞춰줘야 대칭이 된다.

## 게임 루프

### 누적 시간 하나로 모든 시간 기반 연출을 구동

등장 지연·플래시·파티클 sway·펄스는 전부 "시간"이 필요하다. Ticker 콜백에서 dt를 누적한 `_animT` 하나를 공유해, 각 연출이 이 시간축을 기준으로 자기 위상을 계산한다. 위상(phase)은 반복하는 애니메이션에서 "지금 한 주기의 어디쯤 왔나"를 0~1로 나타낸 값이다.

```dart
void _tick(Duration elapsed) {
  final dt = (elapsed - _lastElapsed).inMicroseconds / 1e6;
  _lastElapsed = elapsed;
  _animT += dt; // 등장 지연(_fillBorn)·플래시·sway가 이 시간축을 공유
}
```

시간 기반 연출이 여러 개면 각자 타이머를 두지 말고 하나의 누적 시간을 공유하는 게 관리가 단순하다. 정지·복원도 이 값 하나만 다루면 된다.
