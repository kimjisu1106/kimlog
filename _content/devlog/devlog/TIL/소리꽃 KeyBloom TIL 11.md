소리꽃에 나선 기둥·물고기 유영 모션을 추가하며 배운 것들. 데이터 주도 설계 덕에 새 모션이 switch 케이스 하나로 끝난 것, 파라미터 하나(swayGrow)가 원뿔과 원통을 가른 것, 물고기 헤엄을 만든 위상 동기 추진 맥동, 그리고 "가운데 직진 없는" 랜덤 분포 만들기.

---

## 새 모션이 케이스 하나로 끝나는 이유 — 그리고 파라미터 하나의 차이

소리꽃 파티클은 모션(피어오름·토네이도·분수…)마다 update 코드를 따로 두지 않는다. 모든 모션은 spawn(입자가 태어나는) 시점에 물리값(속도·중력·감쇠·좌우 흔들림·수명)으로 환원되고, update는 그 값으로 도는 하나의 공용 계산기다. 모션마다 코드를 따로 두지 않고 속도·중력 같은 숫자만 받아 입자 위치를 매 프레임 조금씩 갱신하는데, 이런 걸 제네릭 적분기라고 부른다.

```ts
function motionPhysics(mode: MotionMode, speed: number): Physics {
  switch (mode) {
    case "spiral": { /* 물리값 반환 */ }
    case "helix":  { /* 물리값 반환 */ }
    // 새 모션 추가 = 여기 케이스 하나
  }
}
```

이 구조의 힘을 이번에 실감했다. "토네이도랑 비슷한데 반경이 안 퍼지고 원통으로 도는 것"이라는 요구가, 알고 보니 파라미터 하나 차이였다.

```ts
// 토네이도(spiral): 흔들림 진폭이 시간에 따라 커진다 → 위로 갈수록 퍼지는 원뿔
swayGrow: 1.2 + r() * 0.6,

// 나선 기둥(helix): 진폭 성장 0 → 반경이 고정된 원통
swayGrow: 0,
```

여기서 `r()`은 0~1 사이 아무 숫자나 뽑는 주사위 같은 함수다(입자마다 조금씩 다른 값을 주려고 쓴다). 나머지는 스타일 수치다 — 반경(swayAmp)을 절반으로, 회전(swayFreq)을 빠르게, 수명(lifeSec)을 길게 해서 좁고 높은 기둥. 모션의 정체성이 코드가 아니라 데이터에 있으면, 새 모션은 "어떤 물리값이면 그 모양이 나오는가"라는 질문으로 바뀐다.

## 물고기 헤엄 — 꼬리 칠 때 훅 나아가게 (burst-glide)

물고기는 좌우 꼬리짓(S자)만으론 부족했다. 실제 물고기는 꼬리를 칠 때 훅 나아가고 그 사이엔 미끄러진다(burst-glide). 즉 수직 속도가 일정하지 않고 꼬리짓 박자로 맥동한다.

기존 물리엔 이걸 표현할 값이 없어서 필드 한 쌍을 추가했다 — 수직 속도를 사인파로 변조하는 추진 맥동.

```ts
// update() — 수직 추진 맥동. pulseAmp 0이면 기존과 동일
const thrust = 1 + p.pulseAmp * Math.sin(p.pulseFreq * p.age + p.swayPhase);
p.y += p.vy * thrust * dt;
```

포인트는 위상(swayPhase)을 좌우 흔들림과 공유한 것. 위상은 파도가 지금 마루에 있는지 골에 있는지를 가리키는 값이다 — 꼬리짓(sway)과 추진(thrust)이 같은 사인 위상으로 움직이니 "꼬리를 칠 때 나아간다"는 인과가 눈에 보인다. 따로 랜덤 위상을 주면 흔들림 따로 추진 따로 노는 로봇 같은 움직임이 된다.

```ts
case "fish": {
  const tail = 4.5 + r() * 1.5;   // 꼬리짓 주기
  return {
    swayFreq: tail,               // 좌우 꼬리짓
    pulseAmp: 0.4,
    pulseFreq: tail,              // 추진도 같은 주기 (+ 같은 swayPhase)
    // ...
  };
}
```

제네릭 물리에 필드를 추가할 땐 기존 케이스 전부에 0을 채웠다. 기본값 0 = 기존 동작 불변이라, 필드가 늘어도 다른 모션은 아무 영향이 없다.

## 시작 위상을 흩으면 원통 모양이 된다

나선 기둥에서 파티클마다 시작 위상(`swayPhase`)을 랜덤으로 주면, 개체는 전부 같은 규칙(같은 반경·같은 주기)으로 돌지만 서로 다른 지점에 있다. 좌우 사인 궤적이 위상만 다르게 겹치면 — 멀리서 보면 원통 표면을 도는 입자들처럼 보인다.

```ts
swayPhase: r() * Math.PI * 2,  // 파티클마다 위상 분산 → 원통 실루엣
```

개별 파티클의 움직임(1D 사인)과 군집이 만드는 형태(3D 원통 느낌)가 다르다는 게 재미있는 부분. 형태는 개체가 아니라 분포가 만든다.

## "가운데 직진 없는" 랜덤 — 방향을 정하고 최소 속도를 보장

물고기를 처음엔 `vx: (r() - 0.5) * 0.045`로 흩었더니, 0 근처가 뽑힌 물고기는 방향감 없이 애매하게 직진했다. 원하는 건 "왼쪽이든 오른쪽이든 정하면 그쪽으로만"이다.

균등 랜덤은 0 주변도 균등하게 뽑는다. 방향이 있는 분포를 원하면 부호와 크기를 분리해서 뽑아야 한다.

```ts
// ❌ 0 근처(방향 없는 개체)가 섞임
vx: (r() - 0.5) * 0.045,

// ✅ 부호(방향)를 먼저 정하고, 크기는 최소값을 보장
const dir = r() < 0.5 ? -1 : 1;
vx: dir * (0.02 + r() * 0.03),  // 항상 |vx| ≥ 0.02
```

"랜덤이되 어떤 값은 나오면 안 된다"는 요구는 생각보다 흔한데(방향·최소 크기·금지 구간), 분포를 직접 설계하면 조건문 없이 깔끔하게 풀린다.

## 요약

- 데이터 주도 모션 시스템에선 새 모션 = "어떤 물리값이면 그 모양인가". 토네이도와 나선 기둥의 차이는 `swayGrow`(진폭 성장) 하나 — 성장하면 원뿔, 0이면 원통.
- 물고기 burst-glide는 수직 속도를 사인으로 변조(추진 맥동)하되, 좌우 흔들림과 위상을 공유해야 "꼬리 칠 때 나아가는" 인과가 보인다.
- 제네릭 구조에 필드를 추가할 땐 기본값 0 = 기존 동작 불변으로.
- 군집의 형태(원통)는 개체의 궤적이 아니라 위상 분산이 만든다.
- "방향 있는 랜덤"은 부호와 크기를 분리해 뽑고 최소값을 보장 — 0 근처의 애매한 개체를 분포 설계로 제거.
