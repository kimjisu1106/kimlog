메모리 주소: 메모리는 방이 엄청 많은데 변수를 생성하면 해당 변수가 메모리 안에서 방을 하나 배정받는다. 그 방의 위치가 메모리 주소.
포인터: 그 방 번호를 저장하는 변수.

```cpp
int a = 10;
std::cout << &a << std::endl;  // 출력하면 0x00AFBC12 같은 게 나옴. 이게 "a가 살고있는 방 번호"
int* ptr = &a;  // ptr = "a의 방 번호를 저장"
std::cout << *ptr << std:: endl;  // "그 방 번호로 가서 안에 있는 값을 꺼낸다"
```

포인터는 FK나 ID처럼 직접 값을 갖는 게 아니라 어딘가를 가리킨다는 점에서 비슷한 개념. 단, 포인터는 실제 물리적 메모리 주소를 갖는다.

```
- 방 번호(주소) = FK 값 (ex. `3`)
- 방 안에 있는 것 = FK가 가리키는 실제 행의 데이터
- `*ptr`로 접근 = JOIN 해서 데이터 꺼내오는 것
```

메모리 주소는 프로그램 실행할 때마다 달라진다. 그러나 한 번 실행되는 동안에는 고정된다. 따라서 주소를 하드코딩하는 것은 의미없고, 항상 `&a`처럼 그 시점의 주소를 받아서 사용해야 한다.

포인터 변수 사용 시 `*`의 위치

```
int* ptr = &num;   // int 있고 * 있음 → 선언
*ptr = 3000;       // int 없고 * 있음 → 참조 (방 안에 들어가는 것)

int* ptr   →  *가 선언이라는 게 눈에 잘 들어옴
int *ptr   →  문법적으론 동일, 근데 선언인지 참조인지 헷갈릴 수 있음

//다중 선언 시 주의사항
int* a, b;   // a만 포인터, b는 그냥 int
int *a, *b;  // 둘 다 포인터
```
