## 오늘 한 일

- 회사 툴
	- Django에서 context 변수명을 `{{ object.xxx }}` to `{{ obj.xxx }}`로 변경
		- 이유: `object`, `obj`, 고유한 변수명 이렇게 3종류로 context 변수명이 선언되어있었다. `object`는 Python의 내장 함수와 이름이 겹치기 때문에 변경하는 것이 좋다.
		- 해결: 일단 우선순위가 밀려 `object` to `obj`로 변경했고 추후 고유한 변수명으로 변경할 예정. `claude.md`에도 고유 변수명 선언하도록 명시함
	- 연차와 주휴 표기 방식 변경
		- 이유: 연차든 주휴든 둘 다 쉬는거니까 약간 붉은 배경으로 표시해두었는데 팀원들의 구분이되면 좋겠다는 의견 접수
		- 해결: 연차는 좀 더 붉은 배경으로 변경함
	- `DEBUG=False` 및 landing page 생성
		- 이유: 그동안은 혼자 개발 중이니까 `DEBUG=True`로 해두고 에러나면 바로 작업했는데 앱시트 이상으로 팀원들이 바로 사용해야하는 상황이 되어 수정이 필요했다. 사용자는 error를 볼 필요도 없고 굳이 코드를 노출할 필요가 없음.
		- 해결: `DEBUG=False`설정. 사용자용 landing page 및 관리자용 error log page 생성. 500 에러 발생 시 에러 자동 캡처 → DB 저장. 사용자는 landing page로 이동. 관리자에게 누가, 언제, 뭘 할 때 에러가 발생되었는지 로그 List 및 해결 여부 체크. 로그 들어가면 에러 상세 볼 수 있고 clipboard에 copy 할 수 있도록 구현했다. `is_admin`이면 `DEBUG=False`일 때처럼 디버그 페이지 표시되게 함(DB 저장은 동일하게 진행됨).
